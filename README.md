# Project 2: Responsive Web Design

Asher Ruiz: asherr42

## Description:

Project 2: With Responsiveness in mind, I decided to poor my efforts this project into designing an epic index.html or landing page for my website. I figured since it would be the first thing everybody would see, it needed to be great. I utilized as many of the dev tools as possible to give myself the upper hand.

## What I learned:

Responsiveness is a beast. 1 minute you think your page is perfectly responsive, and then you take a look at the Samsung edition 3 and realize that some part of the CSS wasn't perfectly adjusted for everything. Unfortunately, despite my best efforts this time around, I was not able to reach perfection in responsiveness. However, I can take pride in how far I have come, in the fact that now my page adjusts as the user changes the window sizes.

As well, I learned that linking an svg file can make the html file a lot more readable. I think in the future I will do this as I did with my background instead of how I embedded my hero. There's a lot of code that kind of crowds everything.

### Initial Ideas:

The goal of this project was to make my landing page as strong as possible. I wanted it to be simple, responsive, and follow the general guidelines of design.

### Problems I ran into:

Responsiveness - The funny thing is that I designed the webpage starting with mobile in mind first. Yet, when I inspected the many other phones, things did not seem to match up. The furthest I got this time around was making the page responsive based on width. So basically, when you minimize my page from the side, it blends together very well (in my opinion). This also inspired me to change up some of the positioning as the width turned into more of a desktop format.

### How Project evolved:

The focus of my project was learning responsiveness and starting with mobile in mind. I also placed a special emphasis on purposefully finding new ways to design my website based on the dev tools given to us. This led to some crazy new strides in terms of the design of my website. SVG has been a new love of mine, and as a class we should definitely learn more about how they work fundamentally.

### Design and Layout Decisions Made:

A big layout decision made was deciding where to put the header based on the device width. I decided that on mobile devices, I would scale down the size and keep the header at the top. But, as the width grew to be that of maybe a desktop or laptop, I wanted to go for more of an app look by rearranging my Hero with the Header. I thought this gave it more of a formal look.

## Dev Tools Used:

Iconify, Haiku, Image Compressor, Neumorphism/UIverse, Undraw

### Iconify

I used Iconify to grab transparent icons that clearly identified my links to other future tools. I loved this, as it allowed me to match the color scheme I was going for while not adding any crazy background colors (as has happened to me in the past).

### Haiku

I felt my page was lacking something to cut the squareness of it all. Haiku helped me bridge this gap of contrast by making my background soft, helping my words and images to stick out more clearly. I also enjoyed how this background disappears as the page shrinks into a more mobile look. It definitely would have crowded my mobile page had it stayed.

### Neumorphism

I wanted my buttons and links to pop this time around. The first big choice was drawing my user's eye to the button. With a couple adjustments to the pre-set button found on this page, I was able to match it with the theme and color pallette of my page. I was also able to give it a new hover quality that added a gold/yellow pop to my page for when my user is ready to build their budget.

### Undraw

This helped me find my hero. I wanted the page to feel like someone professional was walking the user towards financial health. The guy in the suit pointing at business slides really drove this point across. I will definitely use these in the future!
